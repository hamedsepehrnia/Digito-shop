DJANGO ADMIN PANEL COMPLETE CUSTOMIZATION GUIDE
===============================================

Apply the following comprehensive customizations to Django admin panel:

1. PERSIAN LOCALIZATION
   - Add verbose_name and verbose_name_plural to all model Meta classes in Persian
   - Add verbose_name to all model fields in Persian
   - Add ordering to Meta classes for consistent sorting
   - Add verbose_name to AppConfig classes in Persian

2. CUSTOM ADMINSITE
   - Create a custom AdminSite class that dynamically loads settings from AdminSettings model
   - Implement each_context() method to inject admin_settings into all admin templates
   - Update urls.py to use custom admin_site.urls instead of admin.site.urls

3. ADMIN SETTINGS MODEL (Singleton Pattern)
   - Create AdminSettings model with fields:
     * use_jalali_date (Boolean, default=True)
     * site_title, site_header, site_index_title (CharField)
     * show_hidden_models (Boolean, default=False)
   - Implement singleton pattern: only one record can exist
   - Override save() to prevent multiple records
   - Add has_add_permission() and has_delete_permission() to admin class
   - Create get_settings() classmethod that returns or creates default settings

4. JALALI DATE CONVERSION
   - Create core/admin_utils.py with format_date_for_admin(date_obj, include_time=True) function
   - Function should check AdminSettings.use_jalali_date and convert accordingly
   - Use persiantools.jdatetime.JalaliDateTime for conversion
   - Add jalali_created and jalali_updated methods to all admin classes
   - IMPORTANT: Only show jalali_created in list_display and fieldsets, NOT both created_at and jalali_created
   - Remove created_at from readonly_fields if jalali_created is present

5. HIDDEN MODELS WITH ADVANCED MODE
   - Identify less frequently used models (OrderItem, CartItem, ProductImage, ProductSpecification, PhoneOTP, AboutSection, etc.)
   - Create separate admin classes for hidden models
   - Implement update_hidden_models() function that registers/unregisters based on AdminSettings.show_hidden_models
   - Use admin_site.is_registered() to check before registering/unregistering
   - Connect signal handler in AppConfig.ready() to refresh hidden models when AdminSettings changes

6. SINGLETON MODELS
   - Apply singleton pattern to AdminSettings, ContactInfo, FooterSettings
   - Override save() method: if not self.pk and Model.objects.exists(): return
   - Add has_add_permission() to prevent adding new records
   - Add has_delete_permission() to prevent deletion

7. PERSIAN FONT
   - Create templates/admin/base_site.html
   - Add Vazir font CDN: https://cdn.jsdelivr.net/gh/rastikerdar/vazirfont@v30.1.0/dist/font-face.css
   - Apply font to: body, input, textarea, select, button

8. ADMIN CLASS IMPROVEMENTS
   - Add comprehensive list_display, list_filter, search_fields
   - Add list_editable for frequently changed fields
   - Add readonly_fields for computed values and dates
   - Add proper fieldsets for organization
   - Add date_hierarchy for date-based filtering
   - Set list_per_page (typically 25)
   - Use format_html() for HTML content (status badges, image previews)
   - Add inline admin classes for related models

9. FILE STRUCTURE

   - core/apps.py: Signal handler in ready() method
   - Digito/admin.py: Custom AdminSite, model registration logic
